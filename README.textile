h1. ■フレームワークの思想

役割分担。
１つの小さな処理をうまくこなすコードが協調し合って、全体の仕事を成しとげる。

h2. ■MVCアーキテクチャ

* Model      ... データ出し入れ専門
* View       ... HTML作成専門
* Controller ... 各所に指示して、最後まで面倒を見る役目

h2. ■処理の流れ

# すべてのリクエストを一点に集める（.htaccessへリライト記述しエントリスクリプトindex.php等に集める）
# URI分析し、リクエストされたアクション（コントローラのメソッド）に処理を渡す
# コントローラからモデルへデータ出し入れ指示
# コントローラからビューへHTML作成を指示（完成したHTMLはコントローラに引き渡す）
# コントローラから、HTMLレスポンス
